# EVault バックアップ・サービス入門

バックアップにより、デバイスの外に安全にデータを保管して損失から保護することができます。EVault バックアップは、EVault WebCC ブラウザー・ベースの管理ユーティリティーにより管理される自動エージェント・ベースのバックアップ・システムで、{{site.data.keyword.BluSoftlayer_full}} ネットワーク上の 1 つ以上のデータ・センターにあるサーバー間でデータをバックアップする方式がユーザーに提供されます。管理者は、フル・システムや特定のディレクトリー、あるいは個別のファイルを対象とした毎時スケジュール、日次スケジュール、週次スケジュール、またはカスタム・スケジュールに従うバックアップを設定できます。追加のプラグインにより、Microsoft Exchange、Microsoft SQL といったソフトウェアや、その他のタイプのサード・パーティー・ソフトウェアとの互換性が確保され、ユーザーは必要に応じてベアメタルのリストアを実行できます。

## EVault の注文 

EVault バックアップ・サービスを購入するには、以下の 2 つの方法があります。

- サーバーの注文時における EVault の購入
- アップグレードとしての EVault の購入

### サーバーの注文時における EVault の購入

1. [IBM Cloud カタログ](https://console.bluemix.net/catalog/){:new_window}または [{{site.data.keyword.slportal}}](https://control.softlayer.com/){:new_window}にログインします。
2. 毎月のベアメタル・サーバーを注文します。ベアメタル・サーバーの注文について詳しくは、[ここ](https://console.bluemix.net/docs/bare-metal/baremetal-provision.html){:new_window}を参照してください。
3. 注文を完了するための管理ポータルにリダイレクトされます。<br/>
  **注**: EVault バックアップ・サービスは、時間課金サーバーを注文する際には選択できません。ただし、アップグレードとして後からサービスを追加できます。 
4. **「アドオン」**セクションで、いずれかの EVault オプション (「なし」以外) を選択します。
6. ページの下部にある**「注文に追加」**をクリックします。「ご清算」ページが表示されます。
7. **「清算」**ページで、「ホスト名とドメイン名」セクションを見つけ、ホスト名とドメイン名を入力します。任意のホスト名とドメイン名を選択できます。
8. **「清算」**ページの右側で、**「クラウド・サービスご利用条件 (Cloud Service terms)」**および**「サード・パーティー・サービス契約 (Third-Party Service Agreement)」**チェック・ボックスをクリックします。
9. 支払情報を確認するか入力し、**「オーダーの送信」**をクリックします。プロビジョニング注文番号が表示された画面にリダイレクトされます。これはプロビジョニング注文の受信も兼ねているので、画面を印刷することができます。<br/>**注**: **「見積もりとして保存」**をクリックして購入せずにこの注文を保存することもできます。

プロビジョニング注文の受領確認、プロビジョニング注文の承認と処理、およびプロビジョニングの完了といった一連の E メールが管理者に送信されます。プロビジョニングの完了 E メールには、{{site.data.keyword.cloud_notm}} にログインした後の*「デバイスの詳細」* ページへのリンクが含まれます。また、{{site.data.keyword.slportal}}に直接ログインすることもできます。

**EVault サービス購入の確認**
1. [{{site.data.keyword.slportal}}](https://control.softlayer.com/){:new_window}で、メインメニューから**「デバイス」** > **「デバイス・リスト」**を選択します。 
2. 注文した新規サーバーを見つけます。
  - URL の横に時計アイコンがある場合は、EVault の購入確認に進むまで待つ必要があります。ページを最新表示して、新規サーバーに関する状況が更新されるか確認します。時計アイコンが表示されなくなったら、EVault サービスの購入を確認する以降のステップに進むことができます。
  - 該当のサーバーに対して時計アイコンが表示されなくなったら、リンク (サーバーの URL) をクリックして、**「デバイスの詳細」**ページに進みます。 
3. **「ストレージ」**タブをクリックし、EVault の情報を表示します。
4. 「EVault」セクションを検査し、EVault の購入処理時に選択したサイズが EVault リンクの横に表示されることを確認します。

### アップグレードとしての EVault の購入

**EVault をインストールするサーバーの選択**
1. [IBM Cloud カタログ](https://console.bluemix.net/catalog/){:new_window}または [{{site.data.keyword.slportal}}](https://control.softlayer.com/){:new_window}にログインします。
2. メインメニューから**「デバイス」** > **「デバイス・リスト」**を選択します。EVault サービスを追加するデバイスを検索します。
3. デバイス名をクリックして、「デバイスの詳細」ページに進みます。

**EVault サービスの追加 (購入)**
1. **「ストレージ」**タブをクリックし、ページの下部近くにある「EVault」セクションを見つけます。
2. **「追加」**リンクをクリックします。
3. ポップアップで場所を選択するかデフォルトを受け入れ、サイズを選択します。
4. **「続行」**をクリックします。
5. ご利用条件に同意する場合はチェック・ボックスをクリックします。
6. **「注文」**をクリックします。

**EVault サービス購入の確認**
1. **「デバイスの詳細」**ページを最新表示し、**「ストレージ」**タブが選択されていることを確認します。
2. 「EVault」セクションを検査し、EVault の購入処理時に選択したサイズが EVault リンクの横に表示されることを確認します。<br /> **注**: EVault ストレージ・サイズとして引き続き容量ゼロが表示される場合は、ページの 2 回目のリフレッシュが必要になることがあります。 

## {{site.data.keyword.slportal}}での EVault バックアップ・ストレージの詳細へのアクセスと表示

EVault バックアップ・サービスに関するストレージの詳細は、いつでも {{site.data.keyword.slportal}}を使用して表示することができます。表示できる詳細には、選択した EVault バックアップ・サービスに関連付けられているパスワード、ストレージ・アドレス、および使用量が含まれます。 

1. [{{site.data.keyword.slportal}}](https://control.softlayer.com/){:new_window}で**「EVault バックアップ・ストレージ (EVault Backup Storage)」**画面にアクセスするには、一意の資格情報を使用してログインします。
2. **「ストレージ」**をクリックし、ドロップダウン・リストから**「バックアップ」**を選択します。
2. 目的の EVault バックアップの行の任意の場所をクリックし、そのストレージの詳細を表示します。このビューでは「パスワード」は表示可能ではありません。次のステップに進み、EVault バックアップ・サービスに関連付けられているパスワードを表示します。
3. **「パスワード」**フィールドの横にある**「表示」**チェック・ボックスをクリックし、選択した EVault バックアップ・サービスのパスワードを表示します。

多くの {{site.data.keyword.BluSoftlayer_full}} 製品およびサービスでは、{{site.data.keyword.slportal}}内でのパスワード・ストレージ機能はパスワードの保管またはトラッキングのみのために使用され、{{site.data.keyword.slportal}}内でパスワードに加えられた変更は製品またはサービスに適用されません。これは EVault バックアップには当てはまり_ません_。{{site.data.keyword.slportal}}内で EVault バックアップのパスワードに加える変更は、サービス自体に対して加えられます。サービスに直接影響があることを念頭に置いて変更を行ってください。パスワードをリセットするには、[{{site.data.keyword.slportal}}での EVault バックアップのパスワードの変更方法](/docs/infrastructure/Backup/change-password-evault-backup-service.html)のステップに従ってください。

## EVault バックアップのための WebCentralControl (WebCC) へのアクセス

WebCentralControl (WebCC) は、{{site.data.keyword.BluSoftlayer_full}} が提供するすべての EVault バックアップ・サービスとの対話時に使用されるクライアントです。WebCC は、構成やリストアを含め、EVault バックアップ・サービスを完全に制御することを可能にする、プライベート・ネットワークで実行されるブラウザー・ベースのクライアントです。EVault バックアップのために WebCC にアクセスするには、以下のステップに従ってください。

1. VPN 経由でプライベート・ネットワークにアクセスします。<br/>
    **注**: パブリック・ネットワーク上で WebCC にアクセスすることはできません。WebCC にアクセスするには、VPN 接続を確立する必要があります。
2. [{{site.data.keyword.slportal}}](https://control.softlayer.com/){:new_window}で「EVault バックアップ・ストレージ (EVault Backup Storage)」画面にアクセスします。
3. 目的の EVault バックアップの行の任意の場所をクリックし、ビューを展開します。
3. **「WebCC ログイン」**をクリックし、ブラウザーで WebCC クライアントを開始します。

## WebCC での EVault エージェントの構成

EVault サービスを注文し、サーバーにエージェントをインストールしたら、データのバックアップの作成を開始できます。エージェント、保存スケジュールを構成し、最初のバックアップ・ジョブを開始するには、以下のステップに従ってください。

1. WebCC にログインします。
2. **「すべてのエージェント」** > **「未構成のエージェント (Unconfigured Agents)」**をクリックします。
3. **「これは構成対象の新しいエージェントです (This is a new Agent I would like to configure)」**リンクをクリックします。プロセスを段階的に実行して以下を指定します。
   1. エージェント構成 (Agent configuration) - エージェントの説明を入力し、**「次へ」**をクリックします。
   2. ジョブ・タイプの選択 (Job type selection) - ジョブ名、ジョブの説明、およびバックアップ・ソース・タイプを入力し、**「次へ」**をクリックします。
   3. 選択 (Selection) - ディレクトリーを選択し、**「包含 (Include...)」**をクリックします。
   4. オプション - パスワードを入力します。
   5. スケジュール - **「追加」**をクリックしてスケジュールを作成し、**「次へ」**をクリックします。
   6. デフォルトのボールトを選択し、**「OK」**をクリックします。
   7. **「保存」**をクリックします。
4. 保存スケジュールを作成します。
   1. **「編集」** > **「エージェント設定 (Agent Settings)」**を選択します。
   2. **「追加」**をクリックします。
   3. 保存の詳細を入力します。
   4. **「OK」**をクリックします。
   5. **「保存」**をクリックします。
   **注** - 保存スキームの仕組みについて詳しくは、[ここ](evault-backup-faq.html#how-do-the-retention-schemes-work-)を参照してください。
5. エージェントを実行し、バックアップを開始します。
   1. **「すべてのエージェント」**をクリックしてから、構成したばかりのエージェントを選択します。
   2. **「バックアップの実行 (Run Backup)」**をクリックします。
   3. 「宛先」を確認し、保存スキームを選択します。
   4. **「バックアップの開始 (Start Backup)」**をクリックします。プロセスの稼働中、バックアップの詳細を表示できます。
   5. バックアップが完了したら、**「閉じる」**をクリックします。
   
**注**: Linux での簡単なファイル・レベルのバックアップを構成する方法について詳しくは、[ここ](configure-simple-file-backup-linux.html)を参照してください。

## 次のステップ

WebCC のシステムは完全に文書化されており、アプリケーションのサポートには WebCC 内からアクセス可能です。右上隅にある、青い円内に白の疑問符が示された**「ヘルプ」**をクリックしてください。ポップアップ・ボックスの左側にあるナビゲーション・バーで任意の記事やトピックをクリックすると、詳細情報が表示されます。
